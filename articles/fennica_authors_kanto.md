# Fennica Authors in Kanto

### Introduction

This document provides an example usage of the search_fennica_authors()
function, which links authors from the Fennica catalog to their
respective Kanto IDs in Finto’s ‘Finaf’ vocabulary. The function helps
enrich metadata for bibliographic records.

``` r
suppressMessages(suppressWarnings({
  if (!requireNamespace("devtools", quietly = TRUE)) {
    install.packages("devtools", quiet = TRUE)
  }
  devtools::install_github("fennicahub/finto", quiet = TRUE, force = TRUE)
}))
```

After downloading the package we select the columns `id`, `Title`,
`Author`, `Name_100a` `Code_1000` `Code_7000` `Code_7001`.

``` r
library(finna)
library(finto)
library(dplyr)
library(stringr)
data("fennica_subset",package = "finna")
fennica_subset <- fennica_subset %>% 
  slice_head(n = 20)
knitr::kable(fennica_subset,caption = "few selected fennica data")
```

few selected fennica data

We used the two main columns we used are `Code_1000` and `Code_7001`
which contain the `authorID` digits and extract those digits and use
them to fetch information from finto/kanto. The columns were renamed as
they are originally [MARC 21 format for bibliographical
data](https://marc21.kansalliskirjasto.fi/bib/index.htm) which represent
fields `100/0` and `700/o` So the processed output of the retrieved
finto/kanto data can be found using the provided code below. It provides
extra many columns including the profession related information.

``` r
authors_df <- finto::get_kanto(fennica_subset)
knitr::kable(authors_df, caption = "Authors Data Frame from kanto")
```

### Integrating to fennica

To integrate the processed KANTO author data with your existing Fennica
dataset, you can follow these steps:

1.  **Process the KANTO Data**: Use the `get_kanto` function to retrieve
    and process the author information from the KANTO database.

2.  **Merge with Fennica Data**: Combine the processed KANTO data with
    your Fennica dataset based on a common identifier, such as
    `authorID`.

Here’s how you can achieve this in R:

``` r
# Extract clean 9-digit Author IDs from the 'author_ID' column
fennica_subset <- fennica_subset %>%
  mutate(
    authorID = str_extract(author_ID, "\\d{9}")
  )

# Prepare authors_df by ensuring it has a matching 'authorID' column
authors_df_clean <- authors_df %>%
  distinct(author_ID, .keep_all = TRUE) %>%
  rename(authorID = author_ID)

# Merge on the cleaned 'authorID' field
merged_data <- left_join(fennica_subset, authors_df_clean, by = "authorID")

# Display the merged result
knitr::kable(merged_data, caption = "Merged Fennica + KANTO Data")
```

  `authorID`s from the `fennica_data` and fetches corresponding RDF data
  from the Finto Skosmos API. It returns a cleaned tibble with the
  retrieved data, including profession labels.

- **Merging Datasets**: The
  [`dplyr::left_join`](https://dplyr.tidyverse.org/reference/mutate-joins.html)
  function merges the `fennica_data` with the `kanto_authors` data on
  the `authorID` column. This ensures that all records from
  `fennica_data` are retained, and matching author information from
  `kanto_authors` is added.

**Note**: Ensure that both datasets have the `authorID` column formatted
consistently (e.g., as character strings) to avoid mismatches during the
merge.
